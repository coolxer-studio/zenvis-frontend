第一章　总则

第1条 组织名称

  1. 本组织定名为"酷X众创社区"（以下简称"社区"），对外开展活动统一使用该名称。
  2. 未经社区运营单位书面许可，任何单位或个人不得擅自使用社区名称及相关标识。

第2条 组织性质

  1. 社区是由个人、团队与企业自愿结成，旨在通过聚合多方资源，围绕人工智能、网络安全与监测预警等方向进行市场探索与价值创造的开放式协作众创平台。
  2. 社区不从事任何违法违规经营活动，坚持非营利性协作导向。

第3条 法律地位与责任划分

  1. 本社区为开放式协作众创平台，不具备独立法人资格，暂以"酷X众创社区"品牌开展活动，由指定运营单位开展运营工作（以下简称"运营单位"）。
  2. 社区对外开展的合法活动及产生的民事责任，由运营单位依法承担；社区成员因个人/团队行为导致的法律责任，由成员自行承担。
  3. 运营单位与社区成员间为平等协作关系，运营单位依据本章程履行运营管理职责，不得利用运营优势损害成员合法权益，成员应遵守运营单位基于本章程制定的各项具体管理规则。
  4. 本章程条款内容若与《中华人民共和国民法典》《中华人民共和国知识产权法》《中华人民共和国个人信息保护法》《社会团体登记管理条例》等国家现行法律法规及规范性文件相冲突，以法律法规及规范性文件规定为准；本社区作为开放式协作众创团体，需严格遵守社区团体相关法规要求，坚持非营利性协作导向，不从事非法集会、违规经营等活动，自觉接受相关主管部门的监督管理，规范开展各类协作创作活动。
  5. 本章程未明确约定的事项，适用相关法律法规及行业惯例。

第4条 使命与愿景

  1. 为所有具备创造力的个人、创意团队及初创企业降低创新门槛，提供低成本的想法验证渠道、真实市场反馈机制与价值变现路径。
  2. 保障创造者合法权益，促进创新资源高效整合，推动"创造—验证—交易—再创造"的创新飞轮持续运转，实现个体创造力与市场需求的精准对接。

第二章　价值理念与基本原则

第5条 核心价值理念

  1. 先付出后收益，自愿自主创造，鼓励成员主动投入智力、技术等资源，以实际贡献获取相应权益。
  2. 尊重知识产权，权利属于创造者，明确各类创作成果的产权归属，充分保护创造者的合法权益。
  3. 以市场定义商业价值，产物可由社区运营单位代理推向市场或收购，助力创作成果实现商业变现。
  4. 以共享定义社会价值，未变现产品须以"社区版本"形式免费向公众提供，推动创新成果的广泛传播与二次创新。

第6条 社区运行基本原则

  1. 自愿原则：成员加入、退出社区及参与社区活动均出于自愿，不受任何强制约束。
  2. 平等原则：所有成员不论个人、团队或企业身份，在社区内享有平等的权利、履行平等的义务，不受歧视。
  3. 开放原则：社区向所有认同本章程、具备相应创造力的主体开放，鼓励跨领域、跨身份的协作交流。
  4. 共享原则：社区内的公共资源、未变现创作成果向成员及公众合理共享，推动创新价值最大化。

第三章　社区成员

第7条 成员分类

  1. 社区成员分为三类，分别为个人成员、团队成员和企业成员。
  2. 个人成员指年满18周岁、具备完全民事行为能力，认同本章程、具备相应创造力的自然人。
  3. 团队成员指由多名个人组成、明确责任人，认同本章程、共同开展创作活动的团体。
  4. 企业成员指依法注册、拥有法人资格，认同本章程、愿意参与社区协作、提供相关资源或支持的公司类主体。

第8条 成员权益与义务

成员权益：

  1. 可在社区主页展示个人、团队或企业风采，支持匿名展示；
  2. 有权在社区内发起招募活动，自主制定招募规则及权益分配方案；
  3. 可自愿选择加入社区成员发起的招募活动，参与创作协作；
  4. 有权决定自己贡献的内容的权利归属，依法享有相应的知识产权及收益权；
  5. 可获得社区的认证及英才推荐，受邀参与社区组织的线下沙龙等活动；
  6. 企业成员可与社区建立战略合作关系，获取技术支持及定期的渗透测试等企业服务；
  7. 有权对社区运营工作提出建议、监督运营单位的履职行为，对违规行为进行举报；
  8. 可依法依规使用社区为成员提供的各类共享技术基础设施资源，包括代码托管服务、镜像仓库服务、大模型token、轻量级VPS服务、AI编程账号等，助力开展创作活动。

成员义务：

  1. 严格遵守本章程及社区运营单位制定的各项管理规则，认同社区价值理念；
  2. 诚信参与社区活动，不得提交虚假信息、恶意招募或破坏社区协作秩序；
  3. 保证所提交的代码、数据、模型、报告等内容未侵犯第三方权利，若发生侵权行为，自行承担全部法律责任；
  4. 履行信息保密义务，严格遵守社区信息安全与保密相关规定；
  5. 积极配合社区运营单位的审核、管理工作，主动参与社区协作，维护社区声誉。

第9条 成员入社流程

  1. 申请人需认同本章程及社区价值理念，具备相应的创造力或资源支持能力。
  2. 采用审核制，申请人提交《入社申请表》：个人成员提交个人相关信息，团队成员需由明确的责任人作为代表提交团队及成员相关信息，企业成员提交企业主体资质及相关联系人信息。
  3. 审核工作由社区运营单位设立的审核小组负责，审核小组在收到申请后，在合理期限内完成审核，未通过审核的，应书面说明理由，申请人可申请一次复核。
  4. 申请人通过身份审核后，需签署《知识产权与保密协议》《数据合规承诺书》，明确权利义务。
  5. 签署相关协议后，社区运营单位发放成员认证证书，申请人正式成为社区成员，享有相应权益、履行相应义务。

第10条 成员退出与除名规则

  1. 自愿退出：成员自愿退出社区的，需提前30日向社区运营单位提交书面退出申请，经确认后，完成相关交接手续，即可退出社区；退出后，其已享有的社区权益自动终止，已履行的义务不再追溯。
  2. 强制除名：成员出现重大失信、违法违规、泄露社区核心机密、恶意破坏协作流程、提交侵权内容等行为的，经社区运营单位核实评估通过后，立即予以强制除名，书面通知被除名成员，并公示相关情况；被除名成员不得再次申请加入社区。
  3. 退出/除名后相关处置：
    1. 成员退出或被除名后，其已贡献的开源部分永久保留在社区，不得要求删除；
    2. 其未被投资认购的闭源部分，可要求社区删除，但社区已分发的"社区版本"可继续流通，不得撤销；
    3. 成员退出或除名后，不再享有社区任何权益，也不再履行社区义务，其遗留的相关责任仍由其自行承担。

第11条 成员匿名参与机制

  1. 成员可选择一个符合社区规范的昵称作为匿名身份，参与社区公共活动、发布相关内容，社区尊重成员的匿名意愿，不强制要求公开真实身份。
  2. 涉及资金结算、产权登记、法律纠纷处理等需要核实身份的场景，成员须向社区运营单位提供可验证的实名信息，社区运营单位对成员的实名信息负有严格的保密义务，不得擅自泄露，若因运营单位过错导致信息泄露，需承担相应责任。

第四章　社区运营者

第12条 运营单位选定方式

  1. 社区运营单位由社区成员投票选举产生，选举对象为社区内具有独立法人资格的企业成员。
  2. 选举方案细则由社区筹备组制定，明确选举流程、投票规则、当选条件等，选举过程公开、公平、公正。
  3. 选举后的运营单位信息为社区内部信息，暂不对外披露。
  4. 运营单位任期由选举方案明确，任期届满可参与新一轮选举，连任次数不限。

第13条 运营单位核心工作职责

  1. 负责本章程的解释工作，根据互联网发展、法律法规变化及社区业务发展需要，提出章程修改建议，组织成员开展章程修改表决工作。
  2. 负责社区日常运营管理，包括社区平台技术维护、品牌宣传、外部合作对接等工作，保障社区正常运转。
  3. 负责社区成员的审核、认证、管理工作，处理成员的入社、退出申请，执行成员除名决议。
  4. 负责调解社区内成员之间、成员与运营单位之间的纠纷，对争议事项进行协调处理，无法调解的，引导当事人通过仲裁等合法途径解决。
  5. 负责社区内招募活动、创作项目的合规性形式审核，发现虚假信息、恶意招募等违规行为，及时暂停相关活动并要求整改。
  6. 负责社区技术基础设施的维护、更新，保障共享资源的正常提供。
  7. 负责社区数据的安全管理，妥善保管成员信息、创作成果等相关数据，确保数据合规。
  8. 履行法律法规及本章程规定的其他职责，不得利用运营职权损害社区成员合法权益。

第五章　创造活动及流程

第14条 核心活动主题

  1. 社区创造活动以产品创造、技术研究为主要方向，聚焦人工智能、网络安全与监测预警等领域。
  2. 鼓励成员围绕上述方向开展创新创作、技术研发、产品迭代等相关活动，推动创新成果的转化与应用。

第15条 项目发起与招募规则

  1. 任何社区成员均可在社区平台发布《招募令》，发起创作项目，《招募令》需明确包含项目背景、创作目标、可执行条件、资源需求、贡献核算方式、权益分配方案、项目执行计划等核心内容，确保信息真实、完整、合规。
  2. 招募令有效期由发起方自行设定，但最长不得超过6个月，有效期届满未达到可执行条件的，招募活动自动取消。
  3. 社区运营单位对《招募令》进行合规性形式审核，重点审核信息真实性、是否存在违规内容等，发现虚假信息、违规条款的，有权暂停招募活动，并要求发起方限期整改，整改后仍不符合要求的，撤销该招募令。
  4. 发起方存在恶意招募（包括但不限于虚假招募、骗取成员资源、恶意拖欠权益等）行为的，按本章程第三章成员除名规则处理，取消其发起资格，并予以强制除名。
  5. 招募期满后，若未达到项目可执行条件，招募活动自动取消，发起方需在社区平台公示相关情况，退还已收取的成员投入资源（若有）。

第16条 项目创作与协作管理

  1. 发起方通过招募令整合好所需资源、确定参与成员后，项目进入创作阶段，参与成员需严格按照招募令中的项目执行计划依次推进创作工作。
  2. 项目的具体任务分工、协作方式、进度安排等，由项目内部成员共同协定，并在项目团队内部公示，确保每位成员明确自身职责。
  3. 社区不强制统一提供协作工具，项目团队需自行选择合适的工具，记录代码、文档、数据、测试等各维度的成员贡献，相关记录作为后续产权比例划分、收益分配的重要依据，项目团队需妥善保管相关记录，社区不负责保存记录。

第17条 创作成果产权归属

  1. 创作成果的产权原则上为所有参与创作的成员按贡献比例共同拥有，具体贡献核算方式由项目发起时在《招募令》中明确，未明确的，由参与成员协商确定，协商不成的，可申请社区运营单位调解。
  2. 创作成果的产权可依法进行交易，具体交易形式、交易价格等由产权所有人共同达成一致后确认，交易完成后，需向社区运营单位备案相关交易协议。
  3. 社区运营单位不对产权归属做实质审查，仅负责形式备案，若因产权归属产生纠纷，由产权所有人自行协商或通过合法途径解决，运营单位可根据需要提供必要的协助，但不承担相关法律责任。

第18条 成果投资认购机制

  1. 社区内创作的成果（包括产品、技术、模型等），可通过社区平台展示、路演等方式，获得企业或其他投资主体的投资认购。
  2. 社区运营单位及其关联企业享有同等条件下的优先认购权，同等条件以第三方投资报价为基准，具体认购方式、认购价格、产权让渡比例等，由认购方与产权所有人商议协定。
  3. 认购完成后，产权所有人需按照协议约定，将相应比例的产权让渡给认购方，未让渡的部分仍按社区"社区版本"规则免费向公众公开，不得擅自限制。

第19条 社区版本共享规范

  1. 未被投资认购的创作成果，必须在成果发布之日起30日内，上传可运行、可验证的社区版本（包括二进制文件或容器镜像+完整使用说明），确保社区版本可正常使用。
  2. 社区版本可由公众永久免费下载、分发应用、二次开发（可选开源），但二次开发及使用过程中，须保留原始社区及创作成员的署名，不得擅自删除、篡改署名信息。
  3. 社区版本的原始发起方，应在发布后6个月内提供必要的技术支持和漏洞修复，保障社区版本的稳定性。
  4. 原始发起方逾期未对社区版本进行维护的，社区运营单位可组织其他社区成员参与维护，维护产生的成果归维护者所有，但需保留原始创作成员及社区的署名。
  5. 创作成果的代码是否开源，由创作成员共同决定；若选择开源，须采用经OSI（开放源代码促进会）认证的许可证，确保开源合规。
  6. 社区运营单位拥有对社区版本的"联合冠名权"，可在社区版本中标注社区名称，创作成员不得拒绝。

第六章　投入与收益

第20条 各方成本投入划分

  1. 社区日常运营维护成本由社区运营单位承担，主要包括社区门户站点、项目主页等线上平台的技术维护、服务器租赁、品牌宣传等相关费用。
  2. 社区内发起并执行的各类创作项目，其全部成本（包括人力、技术、资金、物料等）由项目发起方承担，具体承担方式可由发起方与项目参与成员自行商议协定，社区及运营单位不做干涉，也不承担任何项目相关成本。

第21条 收益归属与分配原则

  1. 社区本身产生的品牌收益、合作收益等，归属于社区运营单位所有，用于社区的日常运营、基础设施维护等工作。
  2. 社区内发起的创作项目所产生的全部收益（包括投资认购收益、产品销售收益、技术授权收益等），属于项目团队所有，由项目参与成员内部协商分配，分配原则上按照成员的贡献比例执行，具体分配方式可在项目发起时在《招募令》中明确，未明确的，由成员协商确定，协商不成的，可申请社区运营单位调解。
  3. 收益分配过程中，成员需依法履行纳税义务，相关税费由成员自行承担，运营单位可提供必要的协助。

第七章　知识产权与合规

第22条 知识产权侵权责任约定

  1. 社区成员参与创作、提交的所有代码、数据、模型、报告、产品设计等内容，均须保证未侵犯任何第三方的知识产权（包括著作权、专利权、商标权等），不得提交抄袭、盗用、侵权的相关内容。
  2. 若因成员提交的内容侵犯第三方知识产权，引发侵权索赔、诉讼等法律纠纷的，由该贡献成员按份承担全部法律责任，包括但不限于赔偿金、法律费用、仲裁费用、律师费等，若多个成员共同贡献侵权内容，由各成员按贡献比例承担相应责任。
  3. 社区运营单位对成员提交的内容仅进行合规性形式审核，不承担知识产权实质审核义务，因此产生的侵权责任，由侵权成员自行承担，与运营单位无关。
  4. 发现侵权行为的，社区运营单位有权立即下架相关内容，暂停侵权成员的社区活动，并要求其承担相应责任，情节严重的，予以强制除名。

第23条 数据合规管理要求

  1. 社区成员开展的创作项目，若涉及个人信息、重要数据、跨境数据转移等内容，须提前完成合规评估，制定合规方案，并向社区运营单位备案，备案内容包括数据来源、数据用途、合规评估报告等。
  2. 成员未按要求进行备案，导致项目违反《中华人民共和国个人信息保护法》《数据安全法》等相关法律法规，受到行政处罚或引发法律纠纷的，由项目组自行承担全部责任，与社区及运营单位无关。
  3. 社区运营单位负责社区内相关数据的安全管理，严格遵守数据合规相关规定，不得非法收集、使用、泄露成员及项目相关数据。

第24条 专利、商标与品牌使用规范

  1. 社区成员在创作过程中产生的发明创造、实用新型、外观设计等，申请专利的权利归创作成员所有，专利申请、维护等相关费用由成员自行承担，专利授权后，相关权益归成员所有。
  2. 任何成员使用社区品牌、注册商标、标识等，须事先取得社区运营单位的书面授权，未经授权，不得擅自使用，不得用于违法违规活动，否则需承担相应的侵权责任。
  3. 社区标识、域名、品牌手册等相关知识产权，由社区运营单位持有，运营单位可许可社区成员在非商业活动中合理使用，使用过程中需遵守运营单位的相关规定，不得篡改、滥用社区品牌及标识。

第八章　纠纷解决

第25条 争议协商与仲裁机制

  1. 社区成员之间、成员与运营单位之间，因社区活动、创作项目、产权归属、收益分配等事项产生争议的，应首先由当事人自行协商解决，协商过程中，各方应秉持公平、诚信原则，积极配合，争取达成一致协议。
  2. 协商不成的，任何一方均可提交天津仲裁委员会仲裁，仲裁裁决为终局裁决，对各方当事人均具有约束力，各方需严格履行仲裁裁决结果。

第26条 成员退出资产与成果清算

  1. 个人或团队成员退出社区时，已开源的创作成果部分继续向社区及公众开放，不得要求删除或撤销。
  2. 未被投资认购的闭源创作成果部分，退出成员可与项目其他成员协商，要求删除相关内容，或由其他成员回购其持有的产权份额，协商不成的，可申请社区运营单位调解，或通过仲裁等合法途径解决。
  3. 退出成员已投入的资源，若未形成创作成果或未产生收益，可与相关方协商退还，若已形成成果并产生收益，按收益分配原则分配相应权益后，不再退还已投入资源。

第九章　信息安全与保密

第27条 信息分级保密体系

  1. 社区信息按保密级别分为三级，分别为公开级、社区内部级、核心机密级。
  2. 公开级：包括社区版本、《招募令》、开源代码、社区章程、成员公开风采等，可向公众自由公开。
  3. 社区内部级：包括项目未开源代码、产品文档、商业计划书、用户数据、成员实名信息等，仅可由社区成员及运营单位相关工作人员查阅、使用，不得向社区外部泄露。
  4. 核心机密级：包括投资认购方提供的私有数据、尚未披露的创作成果漏洞细节、社区运营核心数据等，仅可由相关授权人员接触，严格限制传播范围。

第28条 全员保密义务与追责

  1. 所有社区成员、运营单位工作人员，均负有严格的信息保密义务，不得泄露、篡改、滥用社区各类保密信息。
  2. 接触核心机密级信息的成员及工作人员，须额外签署《附加保密协议》，明确保密责任、保密期限等相关要求，严格履行保密义务。
  3. 任何一方违反保密义务，泄露社区保密信息，给社区、其他成员或第三方造成损失的，需赔偿因此造成的全部损失，包括但不限于直接损失、间接损失、法律费用等。
  4. 违反保密义务情节严重的，社区运营单位有权予以强制除名，涉嫌违法犯罪的，移交司法机关处理。

第十章　技术基础设施

第29条 社区线上平台构成

  1. 社区门户站点（www.coolxer.com），作为社区的核心入口，用于成员注册、信息发布、招募活动展示、创作成果展示等；
  2. 相关招募项目及产品的主页（社区级子域名），每个创作项目可拥有独立的主页，用于展示项目详情、进度、成果等相关信息，方便成员协作及公众了解。

第30条 公共共享基础设施资源

  1. 轻量级VPS服务，用于项目部署、测试等；
  2. 代码托管服务，方便成员存储、管理、协作开发代码；
  3. 镜像仓库服务，用于存储、分发容器镜像等相关文件；
  4. AI编程账号，包括Claude Code、Cursor、通义灵码、阿里云百炼Coding Plan等，助力成员提升编程效率；
  5. 大模型token，包括Anthropic Claude、智谱AI、阿里云百炼、Kimi等，为成员的技术研发、模型训练等提供支持。

上述共享资源的使用规则，由社区运营单位制定并公示，成员需严格遵守，违规使用的，运营单位有权暂停其使用权限。

第十一章　附则

第31条 章程修改流程与表决要求

  1. 根据互联网的发展和有关法律法规及规范性文件的变化，或者因社区业务发展需要，社区运营单位有权对本章程的条款提出修改或变更提议。
  2. 运营单位在提出章程修改建议前，需广泛征求社区成员的意见，通过社区平台公示修改建议，明确公示期限，接受成员的反馈和建议。
  3. 章程修改需经过社区所有成员的表决通过，且表决通过的比例需达到全体成员的三分之二以上，表决结果在社区平台公示，公示无异议后，修改后的章程正式生效。
  4. 修改后的章程，对所有社区成员及运营单位均具有约束力。

第32条 社区解散处置规则

  1. 社区运营单位因经营不善无法继续运营、社区使命与愿景无法达成，或者出现其他符合法律规定的解散情形时，有权决定解散社区，并在社区平台公示解散通知，明确解散时间及相关处置安排。
  2. 社区品牌、标识、域名等相关知识产权属于社区运营单位，由运营单位完成相关权益的转移或保留工作，不得擅自处置。
  3. 社区运营单位在解散后，应妥善处理所有社区数据，包括成员信息、创作成果、项目记录等，确保数据安全和合规，避免数据泄露或滥用，必要时可依法销毁相关数据。
  4. 社区解散后，社区内所有发起的创作项目，由项目实际权利人（团队或成员）自行处理，社区及运营单位不再履行管理职责，也不承担相关责任。

第33条 章程生效时间与备案要求

  1. 本章程经社区全体成员表决通过、公示无异议后，正式生效，生效日期以公示载明的日期为准。
  2. 本章程生效后，由社区运营单位负责向相关主管部门备案（若有），备案过程中产生的相关费用，由运营单位承担。
  3. 本章程是社区运营、成员活动的根本依据，所有社区成员、运营单位均需严格遵守，未尽事宜，按相关法律法规及行业惯例处理。
